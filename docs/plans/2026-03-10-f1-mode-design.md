# F1 模式页面设计文档

## 概述

为个人博客添加一个沉浸式的 F1 主题彩蛋页面，通过渐进式触发方式进入，包含 3D 赛道体验、个人喜好展示和 F1 赛历。

## 触发机制

**渐进式彩蛋设计**：

1. **第一层**：快速点击 "F1爱好者" 3 次 → 触发现有的赛车飞过动画（2.5秒）
2. **第二层**：在动画播放期间继续点击 → 进入 F1 专属页面

实现方式：
- 使用状态追踪点击次数和动画状态
- 动画期间额外点击 2-3 次触发页面跳转
- 页面路由：`/f1-mode`（独立页面，可分享链接）

## 视觉风格

### 配色方案
- **主色**：F1 红 `#E10600` - 霓虹发光效果
- **背景**：深黑 `#0a0a0a`
- **辅助色**：
  - 白色 `#ffffff` - 线条、文字
  - 橙色 `#ff6600` - 火焰、高亮
  - 深灰 `#1a1a1a` - 卡片背景

### 霓虹效果
- CSS `box-shadow` 多层发光
- Three.js `MeshBasicMaterial` + 后处理发光
- 线条粗细：2-4px，带发光拖尾

## 页面结构

### Section 1: 3D 沉浸式赛道

**视觉效果**：
- 用户视角位于赛道"内部"中心
- 赛道线条环绕用户（简化版真实赛道轮廓，如蒙扎 Monza）
- 霓虹红色线条在深色空间中发光

**交互设计**：
- 向下滚动 → 赛道线条逐渐"绘制"（模拟赛车行驶轨迹）
- 同时赛道围绕 Y 轴缓慢旋转（每滚动 100px 旋转 10°）
- 绘制完成后显示赛道名称 "MONZA CIRCUIT" + 淡入

**技术实现**：
- React Three Fiber 创建 3D 场景
- `Line` 组件绘制赛道轮廓
- 使用 `useScroll` hook 控制绘制进度和旋转
- 赛道路径：预定义的点数组，使用 CatmullRomCurve3 平滑

### Section 2: 个人喜好

**内容**：
- 最爱车手：名字 + 国旗 emoji + 车号
- 最爱车队：Logo + 名称
- 最爱分站：赛道简化轮廓图 + 名称

**布局**：
- 三列卡片，移动端单列
- 卡片样式：深灰背景、红色边框、霓虹发光效果
- 滚动进入时 Framer Motion 动画

### Section 3: 2025 赛历

**内容**：
- 24 站比赛信息
- 每站：名称、日期、地点、国家
- 已完成 vs 未完成用不同样式区分

**布局**：
- 时间线或垂直列表
- 每站一行：日期 + 分站名称 + 地点
- 未来比赛高亮显示

## 技术栈

| 技术 | 用途 |
|------|------|
| React Three Fiber | 3D 场景渲染 |
| @react-three/drei | RTF 工具库（Line, OrbitControls 等）|
| Framer Motion | 滚动动画、进入动画 |
| Next.js App Router | 独立页面路由 |
| Tailwind CSS | 样式 |

## 性能考虑

1. **代码分割**：React Three Fiber 相关依赖动态导入，仅 F1 页面加载
2. **3D 优化**：
   - 使用 `InstancedMesh` 复用几何体
   - 限制线条点数（< 500 点）
   - 关闭抗锯齿（移动端）
3. **懒加载**：F1 页面独立 chunk，首屏不加载
